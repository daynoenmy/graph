## 最终实现方案

当前方案不使用生成式扩散模型、扩散重建或知识蒸馏，而是在特征空间进行噪声感知图传播：

1. 根据数据集模态生成与原图空间对齐的医学噪声视图。
2. 使用原图与噪声视图的 patch 特征差异估计噪声不确定性。
3. 降低高不确定性节点作为信息源时的传播权重。
4. 使用正常/异常文本锚点估计病灶概率。
5. 对疑似病灶 patch 降低图更新门控，避免病灶被过度平滑。
6. 使用病灶特征保持损失、边界对比损失和双视图预测一致性损失进行训练。
7. 病灶掩码只用于源训练集的损失计算，目标数据集推理不需要掩码。

鲁棒性测试额外使用 `--test_noise_severity` 对主测试输入施加模态相关噪声；`--noise_severity` 仍只控制模型内部用于估计不确定性的辅助噪声视图。两者相互独立。

主要训练参数：

```bat
--noise_severity 0.06
--noise_consistency_weight 0.1
--lesion_preservation_weight 0.1
--boundary_contrast_weight 0.05
```

`train.bat` 默认使用 Brain 的 `full-shot.jsonl` 训练，`test.bat` 默认使用 Liver 的 `full-shot.jsonl` 测试。数据集元数据读取逻辑保持不变，数据路径仍由 `dataset/constants.py` 管理。

## 训练轮次讨论

Brain 的 `full-shot.jsonl` 包含 3714 个样本。按照 `image_batch_size=2` 计算：

- 每轮约 1857 次参数更新。
- 20 轮约 37140 次参数更新。
- 学习率在 16000 和 32000 step 衰减，约对应第 9 和第 18 轮。

因此，20 轮适合作为首轮实验上限。更严谨的做法是在 Brain 源域划分验证集，比较第 5、10、15、20 轮 checkpoint，再固定模型测试 Liver。不能根据 Liver 测试结果选择 checkpoint，否则会引入目标测试集信息泄漏。

当前训练会保存：

- `text_adapter.pth`：文本适配器。
- `image_adapter.pth`：最新图像适配器。
- `image_adapter_1.pth` 至 `image_adapter_20.pth`：各轮图像适配器。

`test.py` 默认加载最新的 `image_adapter.pth`，也可以通过 `--image_checkpoint image_adapter_20.pth` 等参数明确选择某一轮 checkpoint。

- Brain 使用 Rician 噪声和 MRI bias field。
- Liver 使用 CT 信号相关量子噪声。

- 默认通过 `--image_checkpoint image_adapter.pth` 加载最新训练 checkpoint。

现阶段固定内部辅助视图强度：

```bash

```

依次改变测试主输入噪声强度：


## 批量 checkpoint 测试

`test.py` 已支持使用 checkpoint 通配符批量评估所有训练轮次，并按照数字 epoch 顺序而不是文件名字典序执行。`test.bat` 当前使用：

```bat
--image_checkpoint "image_adapter_*.pth"
```

这会测试 `image_adapter_1.pth` 至 `image_adapter_20.pth`，不重复测试与最后一轮内容相同的 `image_adapter.pth`。为了公平比较，各 checkpoint 在相同随机种子下使用完全相同的测试噪声序列。

每个 epoch 的完整结果分别保存为：

```text
test_<dataset>_epoch_<epoch>_<noise_type>_noise_<severity>.csv
```

所有 checkpoint 的平均指标汇总为：

```text
test_<dataset>_all_checkpoints_noise_<noise_type>_<severity>.csv
```

## V2 实施状态

在保留 V1 默认行为和旧 checkpoint 兼容性的前提下，V2 已作为显式实验路径接入代码，并使用独立的 `train_v2.bat`、`test_v2.bat` 与 `ckpt/noise_graph_v2`。

V2 的实际改动包括：

- 训练噪声池包含 additive、magnitude、signal-dependent、multiplicative 和 low-frequency 五种通用强度扰动机制。
- 训练时约 70% 主输入随机加入噪声，约 30% 保持干净。
- 最大噪声强度按照训练进度从最终上限的 30%、60% 逐步增加到 100%。
- 默认生成两个独立辅助噪声视图；它们只估计 patch 不确定性，不再与主特征平均。
- 多视图一致性损失取平均，并用视图间损失方差进行温和均衡，不采用最坏噪声最大化。
- 训练 mask 只用于限制过强噪声造成的局部病灶/背景对比度丢失，测试推理不使用 mask。
- 图传播改为连续 soft graph，并对图投影使用谱归一化；V2 图输入只使用主图特征。
- 测试时默认根据目标数据集选择对应模态噪声，也可以分别指定主输入噪声和内部探针噪声。
- 新 checkpoint 保存架构配置；测试配置与 checkpoint 不匹配时会明确报错，避免静默评估错误模型。

`full-shot.jsonl`、数据集训练/测试解耦方式和 `dataset/constants.py` 路径管理均未修改。完整说明保存在 `V2_ROBUSTNESS.md`。
